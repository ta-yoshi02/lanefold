# 2026-04-15 Lanefold MVP Notes

- Chosen visual direction: warm industrial-cosmic palette with relay-grid motifs, avoiding direct resemblance to common merge-puzzle palettes.
- Tactical hook: lane attacks scale with row depth so lower placements hit harder, making board positioning matter beyond simple merging.
- Turn order adds a fresh pulse spawn after enemy resolution to preserve merge-game pacing without letting new tiles attack on the same turn.
- Initial implementation will use pure TypeScript simulation modules with Phaser as the rendering/input shell for easier balancing and deterministic browser tests.
- Verification sweep covered title, mid-run play, lane breach, restart, and swipe input using Playwright-driven real-browser captures.
- The bundled `web_game_playwright_client.js` reported valid state but wrote black canvas PNGs in this environment under its default capture path; page-level Playwright screenshots rendered correctly and were used as the visual source of truth.
- Browser console stayed clean after adding `public/favicon.svg` and wiring it from `index.html`.
- No live image generation was performed because `OPENAI_API_KEY` is not configured in this workspace. Procedural Phaser shapes remain the active placeholder-art path.
- Enemy-readability pass: replaced the tall floating enemy area with compact per-lane step tracks above the board. Enemy cards now label `HP` separately from `T-` remaining turns, and lane headers summarize the most urgent enemy in that lane.
- Danger communication: lanes shift to watch/warning/critical colors based on turns-to-breach, front enemies get a `KILL` badge at 1-2 turns remaining, and the board/breach boundary strengthens when global danger rises.
- Layout pass: board cells were reduced from 108px to 94px and the move summary was moved below the board at lower visual weight so lane danger and turn count dominate the HUD.
- Combat-rule pass: old combat was per-tile, bottom-up, row-weighted damage with optional overflow to later enemies. New combat resolves one attack per lane using the sum of all visible tile values in that board column and applies it only to the frontmost enemy; events record `laneDamage`, actual applied damage, and HP before/after.
